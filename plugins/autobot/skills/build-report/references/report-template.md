## 템플릿 사용 규칙

1. **문제는 심각도 순서로 정렬**: critical → major → minor → info
2. **수정 대상은 반드시 플러그인 내 경로**: 생성된 앱 코드가 아닌 Autobot 플러그인 파일
3. **수정 제안은 실행 가능해야**: "개선 필요" 같은 모호한 표현 금지. 구체적 프롬프트 변경, 코드 추가, 로직 수정을 기술
4. **잘 동작한 부분도 반드시 기록**: 문제만 나열하면 무엇을 유지해야 하는지 알 수 없음
5. **환경 정보 포함**: 문제 재현과 환경 의존적 이슈 판별에 필요
